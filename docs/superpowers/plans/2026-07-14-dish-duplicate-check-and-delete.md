# Dish Duplicate Check and Delete Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prevent duplicate dish saves and add safe, consistent edit/delete interactions to the home and library pages.

**Architecture:** A pure name-matching module defines exact, normalized, and conservative similar-name rules. The server exposes a preflight endpoint and repeats the check inside create/update requests, while a nullable unique `name_key` index protects concurrent new writes without rewriting existing data. One item DELETE route owns related-plan cleanup and managed Vercel Blob deletion; the page container owns confirmation and refresh state, while focused components handle long-press and swipe gestures.

**Tech Stack:** Next.js 16 App Router, React 19, TypeScript, Drizzle ORM, Turso/libSQL, Vercel Blob, Tailwind CSS, Vitest.

## Global Constraints

- Exact, normalized, and “红烧肉/红烧肉儿” conservative similar matches block saving.
- Editing excludes the current dish ID and preserves `createdAt`, `id`, and `timesCooked`.
- Deletion is permanent: no recycle bin and no undo.
- Deletion removes related meal plans/history, the dish, and project-managed Vercel Blob images.
- External image URLs only lose their database reference.
- Every delete entry requires the same confirmation dialog.
- Do not write test records into the production database; API checks use an isolated database or manual read-only inspection.
- Preserve unrelated working-tree changes in `next-env.d.ts`, `next.config.ts`, `CLAUDE.md`, and `docs/design-prototype.html`.

- [ ] **Step 3: Implement the pure matcher and shared types**

```ts
export type DishDuplicateKind = "exact" | "normalized" | "similar";

export interface DishNameCandidate {
  id: string;
  name: string;
  imageUrl: string | null;
}

export interface DishDuplicateMatch extends DishNameCandidate {
  kind: DishDuplicateKind;
  message: string;
}
```

```ts
const removable = /[\s\p{P}\p{S}]+/gu;

export function normalizeDishName(name: string): string {
  return name.normalize("NFKC").trim().toLocaleLowerCase().replace(removable, "");
}

function withoutErSuffix(value: string): string {
  return value.endsWith("儿") ? value.slice(0, -1) : value;
}

export function findDishNameMatch(
  name: string,
  candidates: DishNameCandidate[],
  excludeId?: string,
): DishDuplicateMatch | null {
  const eligible = candidates.filter((dish) => dish.id !== excludeId);
  const trimmed = name.trim();
  const normalized = normalizeDishName(name);
  const exact = eligible.find((dish) => dish.name.trim() === trimmed);
  const normalizedMatch = eligible.find((dish) => normalizeDishName(dish.name) === normalized);
  const similar = eligible.find((dish) => {
    const candidate = normalizeDishName(dish.name);
    return candidate !== normalized && withoutErSuffix(candidate) === withoutErSuffix(normalized);
  });
  const hit = exact ?? normalizedMatch ?? similar;
  if (!hit) return null;
  const kind = exact ? "exact" : normalizedMatch ? "normalized" : "similar";
  return {
    ...hit,
    kind,
    message: kind === "similar" ? "可能已经记录" : "菜单库已有这道菜",
  };
}
```

```ts
import { del } from "@vercel/blob";

export function isManagedDishBlobUrl(value: string): boolean {
  try {
    const url = new URL(value);
    return url.hostname.endsWith(".public.blob.vercel-storage.com")
      && url.pathname.startsWith("/zhuzhu-canteen/");
  } catch {
    return false;
  }
}

export async function deleteManagedDishBlob(value: string | null): Promise<void> {
  if (value && isManagedDishBlobUrl(value)) await del(value);
}
```
