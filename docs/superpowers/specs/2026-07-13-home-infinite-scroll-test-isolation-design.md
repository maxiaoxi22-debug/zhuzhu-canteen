# 首页无限滚动与测试数据隔离设计

## 首页

首页仍按创建时间倒序。首次渲染 6 道菜，底部哨兵进入视口时增加 6 道，直到展示内存中已获取的全部菜品。加载过程中不重复请求数据库；全部展示后显示“已经到底了”。不支持 IntersectionObserver 的浏览器直接展示全部，保证功能可用。

## 测试隔离

默认 `npm test` 只运行纯逻辑和配置测试，不调用本地 API，不向 Turso 或 Blob 写入数据。现有 API 集成测试保留作为未来独立测试数据库的测试资产，但不在默认命令中运行。项目文档明确禁止用真实数据库执行 API 写入测试。

## 验证

单元测试覆盖 6 条分页递增、总数边界和小数据集。浏览器以手机尺寸验证首页从 6 条自动增长并最终显示全部。验证默认测试、Lint 和生产构建。
