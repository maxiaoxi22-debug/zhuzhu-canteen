# 猪猪食堂 手动测试用例

## 测试环境
- 手机浏览器（Chrome/Safari）访问 `http://192.168.1.109:3000`
- 或电脑浏览器访问 `http://localhost:3000`

---

## 一、主流程测试

### TC-01：拍照记录菜品（完整流程）
1. 打开首页，确认在「菜品记录」tab
2. 点击「拍照记录一道菜」
3. 选择拍照或从相册选择照片
4. 确认照片预览显示正常
5. 在输入框中填写：菜品名称（如"可乐鸡翅"）
6. 选择分类（如"肉类"）
7. 填写食材清单
8. 填写做法步骤
9. 点击「保存到菜单库」
10. **预期**：保存成功，弹窗关闭，菜品出现在最近记录列表中

### TC-02：AI 智能识别（需要有效 API Key）
1. 点击「拍照记录一道菜」→ 选择一张菜品图片
2. 点击「AI 智能识别（可选）」
3. 等待识别完成
4. **预期**：自动填入菜品名称、分类、食材清单、做法步骤
5. 确认后可修改，点击保存

### TC-03：AI 识别失败后的手动输入
1. 选择一张菜品图片
2. 点击「AI 智能识别（可选）」
3. 如果 API Key 无效，**预期**：显示红色错误提示"AI 识别失败，请手动填写。照片已自动保存"
4. 手动填写菜品信息 → 点击保存
5. **预期**：保存成功，菜品出现在菜单库

### TC-04：浏览菜单库
1. 切换到「菜单库」tab
2. **预期**：显示所有已记录的菜品（按分类排列）
3. 点击分类按钮（如"🥩 肉类"）
4. **预期**：只显示该分类的菜品
5. 在搜索框输入菜品名称
6. **预期**：只显示匹配的菜品

### TC-05：查看菜品详情
1. 在菜单库中点击任意菜品
2. **预期**：弹出详情页，显示菜品图片、名称、分类、食材清单、做法步骤
3. 点击「早餐/午餐/晚餐」按钮
4. **预期**：菜品加入今日菜单，弹窗关闭

### TC-06：随机推荐菜品
1. 切换到「今天吃点啥」tab
2. **预期**：随机显示一道菜
3. 点击「换一道」→ **预期**：显示不同菜品
4. 点击分类标签（如"🥬 青菜"）→ **预期**：只从该分类随机
5. 点击「🌅 早餐」按钮 → **预期**：菜品加入今日早餐列表
6. 切换到下方「今日菜单」，**预期**：早餐栏显示刚添加的菜品

### TC-07：查看制作历史
1. 切换到「历史」tab
2. **预期**：显示常做菜品排行（柱状图）+ 最近点菜记录
3. 点击某条记录 → **预期**：跳转到对应的菜品详情

### TC-08：删除今日菜单中的菜品
1. 在「今天吃点啥」→「今日菜单」中
2. 点击菜品右侧的「✕」按钮
3. **预期**：菜品从该餐次中移除

## 二、边界条件测试

### TC-09：新用户首次使用（无数据）
1. 清空数据库后打开应用
2. **预期**：
   - 「菜品记录」tab 显示"还没有记录菜品，快去拍第一张吧 📸"
   - 「菜单库」tab 显示"还没有记录菜品，先去拍一张吧 📸"
   - 「今天吃点啥」tab 随机推荐区显示"该分类暂无菜品 先去拍一张吧 📸"
   - 「历史」tab 显示"还没有点菜记录，去排个菜单吧 📅"
3. 所有 tab 切换正常，无白屏，无报错

### TC-10：空字段校验
1. 点击「拍照记录一道菜」→ 选择照片
2. 不填菜品名称，直接点保存
3. **预期**：保存按钮不显示（因为 `canSave = editName.trim() && file`）
4. 去掉照片，只填菜品名称
5. **预期**：保存按钮不显示

### TC-11：超长文本输入
1. 菜品名称输入 50 个汉字
2. 食材清单输入 50 行
3. 做法步骤输入 50 行
4. 点击保存
5. **预期**：保存成功，数据不丢失不截断

### TC-12：特殊字符输入
1. 菜品名称输入 `红烧排骨 <script>alert(1)</script>`
2. 点击保存
3. **预期**：保存成功（React 默认转义，无 XSS 风险）

### TC-13：重复保存同一菜品
1. 用相同信息连续保存 3 次
2. **预期**：创建 3 条独立记录（不同 ID），菜单库显示 3 条

### TC-14：无分类菜品
1. 新增菜品时，categoryId 填 null 或 0
2. **预期**：保存成功，菜单库中该菜品在"全部"分类下显示，但不属于任何具体分类过滤

### TC-15：空食材/空步骤
1. 新增菜品时，食材和步骤都留空
2. **预期**：保存成功，详情页不显示食材和步骤区域

### TC-16：快速切换 Tab
1. 快速连续切换 4 个 tab（菜品记录→菜单库→今天吃点啥→历史→菜品记录）
2. **预期**：每个 tab 正常渲染，无闪烁、无崩溃

### TC-17：添加已删除的菜品到菜计划
1. 数据库中有菜品 A
2. 将菜品 A 加入今日早餐
3. 在「今日菜单」中删除早餐中的菜品 A
4. 再在「随机推荐」中刷到菜品 A，加入午餐
5. **预期**：菜品 A 正常加入午餐列表

### TC-18：同一餐次添加多个菜品
1. 对午餐连续添加 3 道不同菜品
2. **预期**：3 道菜都显示在午餐栏中

### TC-19：跨天菜单隔离
1. 今天加入菜品到午餐
2. 修改请求参数 date 为昨天，查询昨天菜单
3. **预期**：昨天的菜单不包含今天添加的菜品

### TC-20：菜品已删除但菜计划仍引用
1. 数据库中有菜品 B，已加入今日午餐
2. 直接从数据库删除菜品 B（不通过前端）
3. 刷新「今天吃点啥」→「今日菜单」
4. **预期**：午餐栏中菜品 B 不显示（因为 `getDish` 返回 undefined 时渲染 null）

## 三、API 测试（自动运行）

```bash
# 确保 dev server 在运行
npm run dev

# 另一个终端运行测试
npx vitest run
```

### API 覆盖的端点：
- `GET /api/dishes` - 列表、搜索、分类过滤
- `POST /api/dishes` - 创建（带数据、空数据、超长数据）
- `GET /api/categories` - 分类初始化
- `POST /api/plans` - 创建菜计划
- `GET /api/plans` - 查询菜计划
- `DELETE /api/plans` - 删除菜计划

## 四、已知限制

1. **AI 识别**：需要有效的 Google Gemini API Key（`AIza...` 格式），当前 key 无效
2. **图片上传**：依赖 Vercel Blob，需要有效的 `BLOB_READ_WRITE_TOKEN`
3. **跨设备访问**：需要手机和电脑在同一局域网，使用 `192.168.1.109:3000` 访问
4. **数据持久化**：数据库在 Turso Cloud，需要稳定的网络连接
5. **删除菜品**：暂无删除菜品的前端入口（需手动通过 API）
